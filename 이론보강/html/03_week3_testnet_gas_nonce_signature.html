핵심 개념 요약

비유로 먼저 이해하기

Faucet은 택배를 보내기 전에 우표를 받는 안내 데스크, nonce는 송장 일련번호, gas는 배송 예산, signature는 위조 방지 도장이다. 우표를 못 받은 문제와 이미 접수한 택배가 실패한 문제를 먼저 나눈다.

Faucet

테스트넷에서 gas 비용으로 쓸 테스트 ETH를 받는 준비 단계다. Faucet 실패는 아직 트랜잭션 실행 실패가 아니라, 실행 전에 필요한 잔고 확보 실패다.

Nonce

계정별 트랜잭션 순번이다. 같은 계정의 nonce는 순서대로 처리되며, 중간 번호가 비면 뒤 번호 트랜잭션은 pending 상태로 기다린다.

Gas와 Signature

Gas는 실행 예산이고 signature는 개인키 소유자가 요청을 승인했다는 증거다. 개인키 자체는 전송하지 않고, 서명값과 공개 정보로 보낸 사람을 검증한다.

EIP-1559 초심자 보충

base fee는 네트워크가 요구하는 최소 기본요금이고, priority fee는 검증자에게 주는 팁이다. max fee는 사용자가 이 트랜잭션에 허용한 gas당 최대 지불 한도다. 실제 적용 단가는 보통 min(max fee, base fee + priority fee)로 이해하면 된다. 단, 실제 tip은 max fee - base fee를 넘을 수 없어 priority fee가 잘릴 수 있다.

중요한 점은 잔고 판정이다. 지갑은 보통 value + gas limit * max fee를 기준으로 보수적으로 잔고가 충분한지 본다. 실제 사용액은 더 적을 수 있지만, 전송 전에 최대 가능 비용을 감당할 수 있어야 한다.

초보자 체크리스트

무엇을 눌렀는가

잔고, gas, nonce, 서명, pending 조건을 바꾸고 판별 버튼을 눌러 어느 단계에서 막히는지 확인한다.

무엇이 바뀌었는가

Faucet 실패는 준비 단계 문제이고, tx hash 이후 실패는 gas, nonce, receipt, require 조건을 따로 봐야 한다.

왜 중요한가

오류를 MetaMask 고장으로 묶지 않고 잔고, 네트워크, nonce, gas, 서명, 컨트랙트 조건 중 하나로 좁힐 수 있다.
