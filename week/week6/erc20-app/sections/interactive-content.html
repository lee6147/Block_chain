토큰 전송

transfer(to, amount)를 호출하여 토큰을 전송합니다.

토큰 위임 승인

approve(spender, amount)를 호출하여 다른 주소가 지정된 수량만큼 토큰을 사용할 수 있도록 승인합니다.

보안 주의: approve() 프론트러닝 취약점

allowance를 A → B로 변경할 때, spender가 변경 트랜잭션을 감지하고 기존 A만큼을 먼저 사용한 뒤 새 B만큼 다시 사용할 수 있습니다 (총 A+B 탈취). 이를 프론트러닝 공격이라 합니다.

안전한 패턴:

  • · allowance를 0으로 먼저 초기화한 후 새 값을 설정 (2단계 approve)
  • · increaseAllowance() / decreaseAllowance() 사용
  • · EIP-2612 permit(): 오프체인 서명으로 approve를 대체
// 안전한 2단계 approve 예시 await token.approve(spender, 0); // 1. 먼저 0으로 리셋 await token.approve(spender, newAmount); // 2. 새 값 설정

허용량 조회

allowance(owner, spender)를 호출하여 현재 위임된 수량을 확인합니다.

잔액 조회

위임 전송

transferFrom(from, to, amount)를 호출하여 승인된 수량만큼 토큰을 대신 전송합니다.

토큰 발행

mint(to, amount)를 호출하여 새 토큰을 발행합니다.

⚠️ MyTokenExtended 전용

mint() 함수는 표준 ERC20에 없으며, 오너 권한이 필요합니다. MyTokenExtended 또는 MyTokenMintable 컨트랙트에서만 사용 가능합니다.

토큰 소각

burn(amount)를 호출하여 내 토큰을 영구 소각합니다.

⚠️ ERC20Burnable 확장 필요

burn() 함수는 OpenZeppelin의 ERC20Burnable 확장을 상속한 컨트랙트에서만 사용 가능합니다. 소각된 토큰은 복구할 수 없습니다.

퍼셋(Faucet)이란?

퍼셋은 수도꼭지를 뜻합니다. 정수기에서 물을 받듯이, 테스트용 토큰을 무료로 지급받을 수 있는 스마트 컨트랙트입니다. 단, 남용을 막기 위해 쿨다운 시간이 있어 일정 시간이 지나야 다시 받을 수 있습니다.

TokenFaucet.sol Solidity 
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/IERC20.sol";
import "@openzeppelin/contracts/token/ERC20/utils/SafeERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/utils/ReentrancyGuard.sol";

/**
 * @title TokenFaucet
 * @dev 테스트용 토큰을 주기적으로 지급하는 퍼셋 컨트랙트
 *      SafeERC20으로 transfer 반환값 검증, ReentrancyGuard로 재진입 방어
 */
contract TokenFaucet is Ownable, ReentrancyGuard {
  using SafeERC20 for IERC20;
  IERC20 public token;
  uint256 public claimAmount;
  uint256 public cooldownTime;

  // 주소별 마지막 클레임 시각 기록
  mapping(address => uint256) private _lastClaimed;

  event TokensClaimed(
    address indexed claimant,
    uint256 amount,
    uint256 timestamp
  );

  constructor(
    address tokenAddress,
    uint256 _claimAmount,
    uint256 _cooldownTime
  ) Ownable(msg.sender) {
    token = IERC20(tokenAddress);
    claimAmount = _claimAmount;
    cooldownTime = _cooldownTime;
  }

  /**
   * @dev 퍼셋 현재 잔액 조회
   */
  function faucetBalance() external view returns (uint256) {
    return token.balanceOf(address(this));
  }

  /**
   * @dev 다음 클레임 가능 시각 조회
   */
  function nextClaimTime(address account)
    external view returns (uint256)
  {
    return _lastClaimed[account] + cooldownTime;
  }

  /**
   * @dev 토큰 클레임 - 쿨다운 확인 후 지급
   */
  // CEI 패턴: Checks → Effects → Interactions
  function claim() external nonReentrant returns (bool) {
    // 1. Checks (조건 검증)
    require(
      block.timestamp >= _lastClaimed[msg.sender] + cooldownTime,
      "Faucet: cooldown active"
    );
    require(
      token.balanceOf(address(this)) >= claimAmount,
      "Faucet: insufficient balance"
    );

    // 2. Effects (상태 변경을 외부 호출보다 먼저!)
    _lastClaimed[msg.sender] = block.timestamp;

    // 3. Interactions (SafeERC20으로 반환값 자동 검증)
    token.safeTransfer(msg.sender, claimAmount);

    emit TokensClaimed(msg.sender, claimAmount, block.timestamp);
    return true;
  }

  /**
   * @dev 오너만 클레임 수량 변경 가능
   */
  function setClaimAmount(uint256 amount) external onlyOwner {
    claimAmount = amount;
  }

  /**
   * @dev 오너만 쿨다운 시간 변경 가능
   */
  function setCooldownTime(uint256 time) external onlyOwner {
    cooldownTime = time;
  }

  /**
   * @dev 오너가 퍼셋 잔액 회수
   */
  function withdraw() external onlyOwner nonReentrant {
    uint256 balance = token.balanceOf(address(this));
    token.safeTransfer(owner(), balance);
  }
}

보안 체크포인트

  • · SafeERC20: transfer() 반환값을 자동 검증합니다. 비표준 토큰(USDT 등)도 안전하게 처리합니다.
  • · ReentrancyGuard: nonReentrant modifier로 재진입 공격을 방어합니다.
  • · CEI 패턴: Checks → Effects → Interactions 순서로 상태 변경이 외부 호출보다 먼저 실행됩니다.
