1

ERC20이란?

비유로 이해하기

USB 포트를 생각해 보세요. 예전에는 제조사마다 충전 포트가 달라서 케이블을 여러 개 들고 다녀야 했습니다. 그런데 USB-C라는 표준이 생기면서 하나의 케이블로 모든 기기를 충전할 수 있게 되었습니다. ERC20은 이더리움 토큰의 USB-C 표준입니다.

  • ● 표준이 없으면 지갑마다 토큰을 따로 지원해야 해서 개발자와 사용자 모두 힘들어집니다.
  • ● ERC20 표준을 따르면 MetaMask, Uniswap, 거래소 등 모든 곳에서 자동으로 호환됩니다.
  • ● ERC = Ethereum Request for Comment (이더리움 개선 제안서)

기술적 정의

ERC20은 이더리움 블록체인 위에서 대체 가능한 토큰(Fungible Token)을 구현하기 위한 스마트 컨트랙트 인터페이스 표준입니다. EIP-20으로 공식 등록되어 있습니다.

  • 대체 가능성(Fungibility): 동일한 토큰 간에 가치가 동일하여 교환 가능
  • 표준 인터페이스: 6개의 필수 함수 + 2개의 이벤트로 구성
  • 호환성: 표준을 따르는 모든 DApp이 별도 구현 없이 토큰 연동 가능
// ERC20 표준 인터페이스
interface IERC20 {
    function totalSupply() external view returns (uint256);
    function balanceOf(address account) external view returns (uint256);
    function transfer(address to, uint256 amount) external returns (bool);
    function allowance(address owner, address spender) external view returns (uint256);
    function approve(address spender, uint256 amount) external returns (bool);
    function transferFrom(address from, address to, uint256 amount) external returns (bool);

    event Transfer(address indexed from, address indexed to, uint256 value);
    event Approval(address indexed owner, address indexed spender, uint256 value);
}
2

핵심 함수 6개

totalSupply()

전체 발행량 조회

비유: 한국은행이 발행한 총 화폐량을 조회하는 것과 같습니다. 시장에 존재하는 토큰 총량을 알 수 있습니다.

balanceOf(address)

잔액 조회

비유: 은행 앱에서 내 잔고를 확인하는 것과 같습니다. 특정 지갑 주소의 토큰 잔액을 반환합니다.

transfer(to, amount)

직접 전송

비유: 인터넷 뱅킹으로 직접 계좌이체하는 것과 같습니다. 본인이 직접 다른 주소로 토큰을 보냅니다.

approve(spender, amount)

사용 권한 부여

비유: 자동이체를 등록하는 것과 같습니다. 특정 컨트랙트가 내 토큰을 대신 사용할 수 있도록 허용합니다.

allowance(owner, spender)

승인량 조회

비유: 자동이체 한도를 확인하는 것과 같습니다. spender가 owner 대신 사용할 수 있는 잔여 허용량을 반환합니다.

transferFrom(from, to, amount)

위임 전송

비유: 자동이체가 실행되는 것과 같습니다. approve로 허가받은 spender가 from의 토큰을 to에게 전송합니다.

// 총 발행량 반환 (읽기 전용)
function totalSupply() external view returns (uint256);

// 특정 주소의 잔액 반환
function balanceOf(address account) external view returns (uint256);

// 호출자(msg.sender)에서 to로 amount만큼 전송
// 성공 시 true 반환, 잔액 부족 시 revert
function transfer(address to, uint256 amount) external returns (bool);

// spender가 호출자 대신 사용할 수 있는 한도 설정
// 덮어쓰기 방식 (이전 값 무시)
function approve(address spender, uint256 amount) external returns (bool);

// owner가 spender에게 허용한 잔여 한도 조회
function allowance(address owner, address spender) external view returns (uint256);

// spender가 from의 토큰을 to로 전송 (allowance 차감)
// DeFi 프로토콜의 핵심 메커니즘
function transferFrom(
    address from,
    address to,
    uint256 amount
) external returns (bool);

이벤트 (Transfer, Approval)

비유로 이해하기

이벤트는 은행 거래 내역서와 같습니다. 돈이 오갈 때마다 기록이 남듯이, 토큰이 전송되거나 승인될 때마다 블록체인에 로그가 기록됩니다. MetaMask나 DApp이 이 로그를 읽어 화면을 실시간으로 업데이트합니다.

Transfer 이벤트

토큰이 전송될 때마다 자동으로 기록됩니다.

  • · from: 보낸 주소
  • · to: 받는 주소
  • · value: 전송량

Approval 이벤트

approve() 호출 시 자동으로 기록됩니다.

  • · owner: 허용한 주소
  • · spender: 허용받은 주소
  • · value: 허용량

활용: 프론트엔드에서 contract.on("Transfer", callback)으로 이벤트를 구독하면 전송이 발생할 때마다 UI를 자동으로 업데이트할 수 있습니다.

이벤트는 EVM 로그(Log)로 저장되며, 스토리지보다 훨씬 저렴합니다. indexed 키워드가 붙은 파라미터는 블룸 필터로 빠르게 검색할 수 있습니다. 

// 토큰 전송 시 발생하는 이벤트 (mint: from=0x0, burn: to=0x0)
event Transfer(address indexed from, address indexed to, uint256 value);

// approve() 호출 시 발생하는 이벤트
event Approval(address indexed owner, address indexed spender, uint256 value);

// ethers.js에서 이벤트 구독 예시
contract.on("Transfer", (from, to, value, event) => {
  console.log(`${from} -> ${to}: ${ethers.formatEther(value)} MTK`);
  // UI 잔액 업데이트 로직 실행
});

전체 코드 해설

비유로 이해하기

이 코드는 회사 설립 서류와 같습니다. 회사를 세울 때 정관(사업 목적, 상호명, 자본금)을 작성하듯이, 스마트 컨트랙트 배포 시 토큰의 이름·심볼·초기 발행량을 설정합니다. 한 번 배포하면 변경할 수 없다는 점도 정관과 비슷합니다.

각 줄이 무엇을 의미하는지 쉽게 설명합니다.

1번 줄

라이선스 표시 — 이 코드는 MIT 라이선스로 누구나 사용 가능합니다.

2번 줄

솔리디티 버전 — 0.8.20 이상 컴파일러를 사용해야 합니다.

3번 줄

OpenZeppelin 라이브러리에서 이미 만들어진 ERC20 코드를 가져옵니다. 우리가 직접 6개 함수를 짤 필요가 없습니다!

4번 줄

MyToken이라는 컨트랙트를 ERC20을 상속받아 만듭니다. (상속 = 기존 기능 전부 가져오기)

5번 줄

배포 시 한 번만 실행되는 초기화 함수 — 이름은 "My Token", 심볼은 "MTK"로 설정합니다.

6번 줄

배포자(msg.sender)에게 initialSupply * 10^18개의 토큰을 발행합니다. (18자리는 소수점 단위)

// SPDX-License-Identifier: MIT
// ↑ 라이선스 식별자: 컴파일러 경고 방지 및 오픈소스 라이선스 명시

pragma solidity ^0.8.20;
// ↑ 0.8.20 이상 버전에서만 컴파일 허용 (오버플로우 자동 체크 포함)

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
// ↑ OpenZeppelin의 검증된 ERC20 구현체 임포트
//   totalSupply, balanceOf, transfer 등 모든 표준 함수가 구현되어 있음

contract MyToken is ERC20 {
// ↑ ERC20을 상속받아 모든 표준 기능을 자동으로 획득

    constructor(uint256 initialSupply) ERC20("My Token", "MTK") {
    // ↑ 배포 시 1회 실행되는 생성자
    //   ERC20("My Token", "MTK"): 토큰 이름과 심볼 초기화

        _mint(msg.sender, initialSupply * 10 ** decimals());
        // ↑ _mint: 내부 함수로 새 토큰을 생성하여 msg.sender에게 지급
        //   decimals()는 기본값 18 반환 → 1 토큰 = 1 * 10^18 최소 단위
        //   예: initialSupply=1000 → 1000 * 10^18 wei 단위 토큰 발행
    }
}

핵심 포인트: decimals()가 18을 반환하는 이유는 이더리움의 최소 단위(Wei)와 동일한 정밀도를 갖기 위함입니다. UI에서는 ethers.formatUnits(balance, 18)로 읽기 좋은 형태로 변환합니다.

MyToken.sol 전체 코드

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
contract MyToken is ERC20 {
    constructor(uint256 initialSupply) ERC20("My Token", "MTK") {
        _mint(msg.sender, initialSupply * 10 ** decimals());
    }
}

확장 기능 (mint, burn, pause)

mint (추가 발행)

중앙은행이 화폐를 추가로 발행하는 것과 같습니다. 새 토큰을 만들어 특정 주소에 지급합니다.

burn (소각)

화폐를 없애는 것과 같습니다. 토큰을 영구적으로 삭제하여 총 발행량을 줄입니다. 디플레이션 효과.

pause (일시정지)

긴급 상황에서 모든 거래를 멈추는 비상 브레이크입니다. 해킹이나 버그 발견 시 사용합니다.

Ownable 접근 제어: mint와 pause 같은 강력한 기능은 컨트랙트 소유자(배포자)만 사용할 수 있도록 제한합니다. 자물쇠가 달린 관리자 전용 기능이라고 생각하세요. 

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Burnable.sol";
import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Pausable.sol";
import "@openzeppelin/contracts/access/Ownable.sol";

contract MyTokenExtended is ERC20, ERC20Burnable, ERC20Pausable, Ownable {
    constructor(uint256 initialSupply)
        ERC20("My Token", "MTK")
        Ownable(msg.sender)  // 배포자를 소유자로 설정
    {
        _mint(msg.sender, initialSupply * 10 ** decimals());
    }

    // onlyOwner: 소유자만 호출 가능한 mint 함수
    function mint(address to, uint256 amount) public onlyOwner {
        _mint(to, amount);
    }

    // onlyOwner: 비상 시 모든 전송 정지
    function pause() public onlyOwner {
        _pause();
    }

    // onlyOwner: 정지 해제
    function unpause() public onlyOwner {
        _unpause();
    }

    // ERC20Burnable을 통해 burn(), burnFrom() 자동 제공

    // Pausable 충돌 해결을 위한 override 필요
    function _update(address from, address to, uint256 value)
        internal
        override(ERC20, ERC20Pausable)
    {
        super._update(from, to, value);
    }
}

Ownable — 소유권 관리

Ownable은 컨트랙트에 "소유자" 개념을 추가합니다. onlyOwner 수식어가 붙은 함수는 소유자만 호출할 수 있습니다.

  • transferOwnership(newOwner) — 소유권을 다른 주소로 이전합니다.
  • renounceOwnership() — 소유권을 영구적으로 포기합니다 (주의: 되돌릴 수 없음).
⚠️ 주의: renounceOwnership() 호출 후에는 mint, pause 등 onlyOwner 함수를 영원히 사용할 수 없습니다. 멀티시그 지갑(Gnosis Safe)으로 소유권을 이전한 뒤 사용하는 것이 안전합니다.

ERC20Capped — 최대 발행량 제한

ERC20Capped를 상속하면 총 공급량이 설정한 한도를 초과할 수 없습니다. 비트코인의 2,100만 개 제한처럼 희소성을 코드로 보장할 수 있습니다. 

import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Capped.sol";

contract MyTokenCapped is ERC20Capped, Ownable {
    constructor()
        ERC20("My Token", "MTK")
        ERC20Capped(1_000_000 * 10 ** 18)  // 최대 100만 개
        Ownable(msg.sender)
    {}

    function mint(address to, uint256 amount) public onlyOwner {
        _mint(to, amount);  // cap 초과 시 자동으로 revert
    }
    // ERC20 + ERC20Capped 상속 충돌 해결
    function _update(address from, address to, uint256 value)
        internal override(ERC20, ERC20Capped) {
        super._update(from, to, value);
    }
}

EIP-2612 Permit — 가스리스 approve

기존 approve()는 사용자가 직접 트랜잭션을 보내야 해서 가스비가 필요합니다. ERC20Permit을 사용하면 오프체인 서명(EIP-712)만으로 approve를 대신할 수 있어, 사용자가 ETH 없이도 토큰을 사용할 수 있습니다 (가스 대납 패턴). 

import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Permit.sol";

contract MyTokenPermit is ERC20, ERC20Permit {
    constructor() ERC20("My Token", "MTK") ERC20Permit("My Token") {}
}
