# 학생 제출물 검증 체크리스트

제출 양식에서 받은 주소/해시를 Etherscan에서 확인해 채점한다.

## 입력값 (제출양식에서)

- `TOKEN_ADDRESS` — 학생의 ERC20 주소 (또는 HNL 재사용)
- `STAKING_ADDRESS` — 학생의 스테이킹 컨트랙트 주소
- `STAKE_TX_HASH` — stake 트랜잭션 해시
- `CLAIM_TX_HASH` — claimReward 트랜잭션 해시
- `STUDENT_WALLET` — 학생 지갑 주소

## 체크 항목

### ① 배포·소스 검증

| 항목 | 방법 | 점수 |
|---|---|---|
| `STAKING_ADDRESS`가 Sepolia에 존재 | Etherscan에서 페이지 로드 | 10 |
| 소스 Verify 완료 | Etherscan **Contract** 탭에 ✅ 뱃지 | 10 |
| 생성자 인자의 `token_`이 `TOKEN_ADDRESS`와 일치 | Contract → **Contract Creation** 데이터 디코드 | 10 |

### ② `Staked` 이벤트

`https://sepolia.etherscan.io/tx/<STAKE_TX_HASH>` → Logs 탭:

- 이벤트 이름: `Staked`
- `user` 토픽이 `STUDENT_WALLET`
- `amount > 0`

통과 시 **25점**.

### ③ `RewardPaid` 이벤트

`https://sepolia.etherscan.io/tx/<CLAIM_TX_HASH>` → Logs 탭:

- 이벤트 이름: `RewardPaid`
- `user` 토픽이 `STUDENT_WALLET`
- `amount > 0`

통과 시 **25점**.

### ④ 잔액 변화 스팟 체크 (선택)

`TOKEN_ADDRESS.balanceOf(STUDENT_WALLET)` — stake 전후 차이.
정확히 일치하지 않아도 *대략* 맞으면 OK (시점 차이로 보상이 추가 적립됐을 수 있음).

### ⑤ 제출 양식 완성도

| 항목 | 점수 |
|---|---|
| 이름/학번 기재 | 5 |
| 모든 주소·해시 기재 | 5 |
| 보상 수식 설명 한 줄 이상 | 5 |
| 소감 한 문단 이상 | 5 |

## 합격 기준

- 기본 합격: **70점 이상**
- 우수: **90점 이상** — 본인 토큰 직접 배포 + Daily 버전 배포 + 소감 우수

## 불합격 사유 예시

- Staking이 **Remix VM**에서 배포됨 → Tx 해시가 Etherscan에 없음 → 재제출 요청
- 이벤트 이름이 `Staked` 아닌 임의 이름 (예: `Deposit`) → 감점
- `pendingReward` 미구현 → `claimReward` 항상 revert → **RewardPaid 이벤트 없음** → 재제출 요청
- 소스 verify 누락 → 조교가 로직을 확인 불가 → 재verify 요청

## 일괄 확인 팁

학생 수가 많다면 Etherscan API로 자동화 가능:

```
https://api-sepolia.etherscan.io/api?module=logs&action=getLogs
  &address=<STAKING_ADDRESS>
  &topic0=0x<keccak256("Staked(address,uint256)")>
  &apikey=<KEY>
```

topic0 해시는 `cast keccak "Staked(address,uint256)"` 또는 online keccak 계산기로 산출.
