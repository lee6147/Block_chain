기본 구조
pragma solidity ^0.8.0 컴파일러 버전 지정. ^0.8.0 = "0.8.0 이상, 0.9.0 미만"의 컴파일러를 사용하라는 뜻
contract Faucet { } 스마트 컨트랙트 선언. 블록체인에 배포되어 자동 실행되는 프로그램. 한번 배포하면 코드 변경 불가
constructor() 생성자. 컨트랙트 배포 시 딱 한 번만 실행. 여기서는 배포자를 owner로 설정
상태 변수 (블록체인에 영구 저장)
address payable owner address = 이더리움 주소 타입 (20바이트). payable = 이 주소로 ETH 전송 가능. public = 누구나 조회 가능
uint256 constant uint256 = 부호 없는 256비트 정수. constant = 배포 후 값 변경 불가 (가스비 절약)
0.01 ether / 1 days Solidity 내장 단위. ether = ×10¹⁸ wei 자동 변환. days = ×86400초. (seconds, minutes, hours, weeks도 가능)
mapping(address => uint256) 키-값 저장소 (Python dict, JS Map과 유사). "주소를 넣으면 숫자를 돌려줌". 초기값은 모두 0
핵심 키워드
require(조건, "에러") 조건이 false면 트랜잭션 즉시 취소(revert) + 가스비 환불 + 에러 메시지 반환. 가장 중요한 보안 장치
modifier onlyOwner() 함수 실행 전 조건 검사 장치. _; 위치에서 원래 함수 코드가 실행됨. 여러 함수에서 재사용 가능
msg.sender / msg.value msg.sender = 함수를 호출한 사람의 주소. msg.value = 함께 보낸 ETH 양 (wei 단위)
block.timestamp 현재 블록이 생성된 시각 (유닉스 타임스탬프, 초 단위). 쿨다운 계산에 사용
external / view / payable external = 외부에서만 호출 가능. view = 읽기 전용 (가스비 무료). payable = ETH를 받을 수 있는 함수
함수 동작 흐름
withdraw() 쿨다운 확인 → 잔액 확인 → 시각 기록 → ETH 전송 → 이벤트 발생. Faucet의 핵심 기능
deposit() Owner만 호출 가능. Faucet에 ETH를 충전. payable이므로 ETH를 받을 수 있음
withdrawAll() Owner만 호출 가능. Faucet의 전체 잔액을 한번에 회수하는 비상 함수
transfer() ETH를 보내는 함수. 실패 시 자동으로 revert (안전한 전송 방법)
emit Event() 이벤트 로그를 블록체인에 기록. DApp 프론트엔드에서 구독(listen)하여 실시간 알림 수신 가능
receive() 누군가 컨트랙트 주소로 직접 ETH를 보낼 때 자동 호출. 없으면 직접 전송 거부됨
