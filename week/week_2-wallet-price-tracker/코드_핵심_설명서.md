## 전체 구조

```
┌──────────── 사용자 브라우저 ────────────┐
│  MetaMask(지갑) ◄► index.html          │
│                    fetch()로 서버 호출   │
└────────────────────┬───────────────────┘
                     │ HTTP
         ┌───────────▼───────────┐
         │  server.js (Node.js)   │
         │  /api/prices → 가격    │
         │  /api/block-info      │
         │  /api/tx → TX 조회    │
         │  /api/send-eth        │
         │  /api/wallet-info     │
         └───┬──────────┬───────┘
             ▼          ▼
        CoinGecko    Infura
         (가격)    (블록체인)
```

> [!tip] 비유
> 레스토랑: `index.html` = 홀, `server.js` = 주방, CoinGecko = 식재료 업체, Infura = 블록체인 출입구, MetaMask = 손님 지갑.

> [!note] 정석
> **클라이언트-서버 아키텍처**. 프론트엔드는 `fetch()`로 백엔드에 HTTP 요청 → 백엔드는 외부 API를 호출 → JSON 응답. MetaMask는 `window.ethereum`을 브라우저에 주입하여 이더리움과 직접 통신한다.

## STEP 1 — 서버 기초

### 모듈과 정적 파일 서빙

```js
var http = require('http');     // HTTP 서버
var https = require('https');   // 외부 API 호출
var fs = require('fs');         // 파일 읽기
var path = require('path');     // 경로 처리
var ethers = require('ethers'); // 이더리움 라이브러리 (npm 설치)
```

서버는 API 요청이 아니면 **정적 파일을 서빙**한다:

```js
function serveStaticFile(req, res) {
  var urlPath = req.url.split('?')[0];
  if (urlPath === '/') urlPath = '/index.html';
  var filePath = path.join(__dirname, urlPath);

  // 보안: ../../ 디렉토리 트래버설 방지
  if (!filePath.startsWith(__dirname)) { res.writeHead(403); res.end('Forbidden'); return; }

  fs.readFile(filePath, function (err, data) {
    if (err) { res.writeHead(404); res.end('404 Not Found'); return; }
    var contentType = MIME_TYPES[path.extname(filePath).toLowerCase()] || 'application/octet-stream';
    res.writeHead(200, { 'Content-Type': contentType });
    res.end(data);
  });
}
```

> [!note] 정석
> `Content-Type`(MIME 타입)은 브라우저에게 파일 종류를 알려준다. `startsWith(__dirname)` 검사로 서버 밖 파일 접근을 차단한다.

## STEP 2 — CoinGecko API 프록시

서버 프록시를 쓰는 이유: ① API 호출 제한 관리 (30초 캐시로 분당 2회로 절감) ② CORS 안정성 ③ API 키 보호.

### 캐시 + API 호출

```js
var priceCache = { data: null, timestamp: 0 };
var CACHE_DURATION = 30000; // 30초

function fetchCoinGecko(callback) {
  var now = Date.now();
  // 캐시 유효 → 즉시 반환
  if (priceCache.data && (now - priceCache.timestamp) < CACHE_DURATION) {
    callback(null, priceCache.data); return;
  }
  https.get({
    hostname: 'api.coingecko.com',
    path: '/api/v3/simple/price?ids=bitcoin,ethereum&vs_currencies=usd,krw...',
    headers: { 'User-Agent': 'BlockchainLab2/1.0' },
  }, function (apiRes) {
    var body = '';
    apiRes.on('data', function (chunk) { body += chunk; });
    apiRes.on('end', function () {
      var json = JSON.parse(body);
      priceCache = { data: json, timestamp: Date.now() };
      callback(null, json);
    });
  });
}
```

> [!tip] 비유
> 메모장에 적어두기. 30초 이내 같은 질문이면 전화(API) 없이 메모를 보여준다.

> [!note] 정석
> HTTP 응답은 **스트림**으로 도착 → `data`로 조각을 모으고 `end`에서 파싱. `callback(err, data)`는 Node.js **에러 우선 콜백** 패턴.

## STEP 3 — Infura JSON-RPC로 블록체인 접속

### Infura란?

이더리움 노드를 직접 운영하면 500GB+ 디스크, 수일간 동기화가 필요하다. **Infura는 클라우드에서 노드를 대신 운영**해주는 서비스(NaaS)다. 우리는 API URL에 요청만 보내면 된다.

### JSON-RPC 요청

```js
function infuraRPC(method, params, callback) {
  var postData = JSON.stringify({ jsonrpc: '2.0', method, params, id: 1 });
  var req = https.request(options, function (apiRes) {
    var body = '';
    apiRes.on('data', function (chunk) { body += chunk; });
    apiRes.on('end', function () { callback(null, JSON.parse(body)); });
  });
  req.write(postData);
  req.end();
}
```

> [!tip] 비유
> JSON-RPC = 전화로 함수 호출. "이더리움씨, `eth_blockNumber` 해주세요" → "`0x134a3b2`입니다."

### 블록 정보 조회 (2단계)

```js
function fetchBlockInfo(callback) {
  infuraRPC('eth_blockNumber', [], function (err, result) {
    var blockHex = result.result;              // "0x134a3b2" (16진수)
    var blockNumber = parseInt(blockHex, 16);  // 20,226,994 (10진수)
    infuraRPC('eth_getBlockByNumber', [blockHex, false], function (err2, blockResult) {
      var block = blockResult.result;
      callback(null, {
        blockNumber, transactionCount: block.transactions.length,
        timestamp: parseInt(block.timestamp, 16),
        gasUsed: parseInt(block.gasUsed, 16), miner: block.miner,
      });
    });
  });
}
```

> [!note] 정석
> 이더리움 JSON-RPC는 **모든 숫자를 16진수 문자열**로 반환한다. EVM이 256비트 정수를 쓰고 JS Number(64비트)로는 정확히 표현 불가. `parseInt(hex, 16)`으로 변환.

## STEP 4 — ethers.js 트랜잭션 조회 & ETH 전송

### Provider와 Wallet 설정

```js
// Provider = 블록체인 읽기 전용 연결
var sepoliaProvider = new ethers.JsonRpcProvider('https://sepolia.infura.io/v3/' + API_KEY);
var mainnetProvider = new ethers.JsonRpcProvider('https://mainnet.infura.io/v3/' + API_KEY);

// Wallet = Provider + 개인키 → 읽기 + 서명(쓰기) 가능
var wallet = new ethers.Wallet(PRIVATE_KEY, sepoliaProvider);
```

| | Provider | Wallet (Signer) |
|---|----------|-----------------|
| 비유 | 읽기 전용 전화선 | 도장이 달린 전화선 |
| 권한 | 잔액/블록/TX 조회 | 위 + 트랜잭션 서명 및 전송 |
| 개인키 | 불필요 | 필요 |

### 트랜잭션 조회

```js
async function fetchTransaction(txHash) {
  var tx = await mainnetProvider.getTransaction(txHash); // 메인넷 먼저
  var provider = mainnetProvider, network = 'Mainnet';
  if (!tx) { // 없으면 Sepolia
    tx = await sepoliaProvider.getTransaction(txHash);
    provider = sepoliaProvider; network = 'Sepolia Testnet';
  }
  if (!tx) return null;
  var receipt = await provider.getTransactionReceipt(txHash);
  return {
    hash: tx.hash, network, from: tx.from, to: tx.to,
    value: ethers.formatEther(tx.value) + ' ETH',
    gasPrice: ethers.formatUnits(tx.gasPrice, 'gwei'),
    status: receipt ? (receipt.status === 1 ? '성공' : '실패') : '대기중',
  };
}
```

> **Transaction**(택배 발송장) = 보낸 순간 생성. **Receipt**(배달 확인서) = 블록 포함 후 생성, 실제 가스·성공 여부 포함.

### Sepolia ETH 전송

```js
async function sendSepoliaETH(toAddress, amountETH) {
  var tx = await wallet.sendTransaction({
    to: toAddress,
    value: ethers.parseEther(amountETH),  // ETH 문자열 → Wei BigInt
  });

  var receipt = await tx.wait();  // 블록에 포함될 때까지 대기

  return {
    hash: tx.hash, from: wallet.address, to: toAddress,
    amount: amountETH + ' ETH',
    blockNumber: receipt.blockNumber,
    status: receipt.status === 1 ? '성공' : '실패',
    sepoliaLink: 'https://sepolia.etherscan.io/tx/' + tx.hash,
  };
}
```

> [!note] 정석
> `ethers.parseEther('0.001')` → `1000000000000000n` (10^15 Wei). 이더리움은 소수점 오류를 방지하기 위해 모든 금액을 **Wei(정수)** 단위로 처리한다. 1 ETH = 10^18 Wei.

## STEP 5 — 거래 원리: 트랜잭션이 처리되는 과정

> [!info] 이 섹션이 핵심이다
> STEP 4에서 `wallet.sendTransaction()` 한 줄로 ETH를 보냈다. 그 한 줄 뒤에서 **실제로 무슨 일이 벌어지는지** 전체 과정을 설명한다.

### 5-1. 전체 흐름

```
사용자 (브라우저)
    │  POST /api/send-eth { to, amount }
    ▼
서버 (server.js)
    │  ① TX 객체 생성 (nonce, gas, chainId 자동)
    │  ② Private Key로 ECDSA 서명
    ▼
Infura → ③ eth_sendRawTransaction 브로드캐스트
    ▼
멤풀 → ④ 수천 TX와 대기 (gasPrice순 정렬)
    ▼
검증자 → ⑤ 서명·잔액·nonce 검증 → 블록 포함
    ▼
블록체인 기록 (불변) → ⑥ receipt 반환
```

### 5-2. 트랜잭션 객체 — 우리가 쓴 것 vs 자동 채워진 것

우리 코드에서는 `to`와 `value`만 지정했지만, ethers.js가 나머지를 **자동으로 채운다**:

| 필드 | 값 (예시) | 누가 채우나 | 설명 |
|------|----------|-----------|------|
| `to` | `0x24F3...` | 우리 | 받는 주소 |
| `value` | `10^15 Wei` | 우리 | 전송 금액 |
| `nonce` | `0` | 자동 | 이 지갑의 몇 번째 TX인지 |
| `gasLimit` | `21000` | 자동 | 최대 가스 사용량 |
| `gasPrice` | `~0.12 Gwei` | 자동 | 가스 1단위당 가격 |
| `chainId` | `11155111` | 자동 | Sepolia 네트워크 |

#### Nonce — 리플레이 공격 방지

은행 거래 **일련번호**. 0번부터 순서대로 매기며, 한 번 쓴 번호는 재사용 불가 → 같은 TX를 복사 재제출해도 거부.

#### Gas — 수수료 계산

수수료 = gasUsed × gasPrice = 21,000 × 0.12 Gwei ≈ 0.0000026 ETH

| 개념 | 비유 | 설명 |
|------|------|------|
| gasLimit | 연료 탱크 | 최대 사용 가능량 |
| gasPrice | 주유비 단가 | 높을수록 빨리 처리 (검증자 우선 선택) |
| gasUsed | 실제 사용 연료 | 단순 전송 = 항상 21,000 gas |

> [!note] 정석
> Gas는 EVM 연산 비용 단위. `ADD` = 3 gas, `SSTORE` = 20,000 gas.

#### Chain ID — 네트워크 구분

서명에 Chain ID 포함 → 메인넷(1) TX를 Sepolia(11155111)에서 재사용 불가 (**EIP-155**).

### 5-3. 디지털 서명 — ECDSA

```
트랜잭션 데이터 → Keccak-256 해시 → Private Key로 ECDSA 서명 → (v, r, s)
```

**키의 관계 (한 방향으로만 가능):**

```
Private Key (256비트 난수, 비공개)
    │  타원곡선 연산 (secp256k1) ← 역산 불가능
    ▼
Public Key (512비트)
    │  Keccak-256 해시 → 마지막 20바이트
    ▼
Address (0x로 시작하는 42자리)
```

| 변환 | 가능? | 이유 |
|------|-------|------|
| Private → Public | ✅ | 타원곡선 점 곱셈 |
| Public → Private | ❌ | 이산 로그 문제 (수학적 불가) |
| 서명 → Public 복원 | ✅ | ecrecover 함수 |

**서명이 증명하는 3가지:** ① **인증** — Private Key 소유자가 만듦 ② **무결성** — 데이터 미변조 ③ **부인 방지** — 부인 불가.

> [!important] Private Key는 네트워크로 전송되지 않는다
> 전송되는 것: TX 데이터 + 서명(v, r, s). Private Key 자체는 **절대 전송되지 않는다.**

### 5-4. 멤풀 → 검증 → 블록 포함

**멤풀**: 블록 미포함 TX 대기열. gasPrice 높은 TX가 우선 선택.

**검증자가 확인하는 5가지:**

| # | 항목 | 확인 내용 | 실패 시 |
|---|------|----------|---------|
| ① | 서명 | from 주소와 일치? | 거부 |
| ② | 잔액 | value + gas비 ≤ 보유량? | 거부 |
| ③ | Nonce | 순번이 맞는가? | 대기/거부 |
| ④ | Gas | gasLimit 충분? | 실패 (gas 소모!) |
| ⑤ | Chain ID | 올바른 네트워크? | 거부 |

> [!note] PoS (지분증명)
> 이더리움은 2022년 The Merge 이후 PoS. 검증자는 32 ETH 예치, 무작위 선출. 부정행위 시 Slashing.

**블록에 포함되면 (상태 전이):**

```
보내는 사람: 잔액 0.050 → 0.04900 ETH (−0.001 ETH − gas비 약 0.0000025), nonce 0 → 1
받는 사람:   잔액 +0.001 ETH
```

이 상태 변경은 **되돌릴 수 없다** — 잘못된 주소로 보낸 ETH는 회수 불가.

> [!warning] 실패해도 가스비는 소모된다
> TX 실패(status: 0)해도 가스비 반환 불가 — 검증자가 연산을 수행했기 때문.

## STEP 6 — 라우팅과 요청 처리

```js
var server = http.createServer(function (req, res) {
  var urlPath = req.url.split('?')[0];
  if (urlPath === '/api/tx' && req.method === 'GET') {
    var hash = new URL(req.url, 'http://localhost').searchParams.get('hash');
    fetchTransaction(hash).then(function (data) {
      res.writeHead(200, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify(data));
    });
    return;
  }
  if (urlPath === '/api/send-eth' && req.method === 'POST') {
    parseBody(req, function (err, body) {
      sendSepoliaETH(body.to, body.amount).then(function (result) {
        res.end(JSON.stringify(result));
      });
    });
    return;
  }
  // /api/wallet-info, /api/block-info, /api/prices도 동일 패턴 → 그 외 정적 파일
  serveStaticFile(req, res);
});
server.listen(5500);
```

> [!note] 정석
> **라우팅**: URL 경로 + HTTP 메서드로 실행 함수 결정. Express 없이 `if`문 직접 구현. GET = 조회(쿼리스트링), POST = 변경(body).

## STEP 7 — 프론트엔드: MetaMask 지갑 연결

> 여기서부터는 `index.html` 내의 `<script>` 코드 (브라우저에서 실행).

### MetaMask 감지 → 연결 → 정보 조회

```js
function initWallet() {
  if (window.ethereum) { onWalletDetected(); return; }
  // 이벤트 리스너 / EIP-6963 / 100ms 폴링(최대 5초) 으로 감지
}

async function connectWallet() {
  var provider = new ethers.BrowserProvider(window.ethereum); // MetaMask 래퍼
  await provider.send('eth_requestAccounts', []);             // 사용자 승인 (EIP-1102)
  await updateWalletInfo(provider);
}

async function updateWalletInfo(provider) {
  var signer = await provider.getSigner();
  var address = await signer.getAddress();
  var balanceEth = ethers.formatEther(await provider.getBalance(address));
  var chainId = Number((await provider.getNetwork()).chainId);
  // 주소, 잔액, 네트워크 표시 / Sepolia(11155111)가 아니면 전환 안내
}
```

> [!note] 정석
> `getSigner()`는 MetaMask에 서명을 **위임**하는 Signer — 개인키가 DApp JS에 노출되지 않는다(핵심 보안 설계). `BrowserProvider`는 **브라우저 내 지갑** 경유, `JsonRpcProvider`는 **서버-Infura** 직접 연결.

### 계정/네트워크 변경 감지

```js
window.ethereum.on('accountsChanged', function (accounts) {
  if (accounts.length === 0) disconnectWallet();
  else updateWalletInfo(new ethers.BrowserProvider(window.ethereum));
});
window.ethereum.on('chainChanged', function () {
  window.location.reload();
});
```

> 초인종 시스템 — 우리가 물어보는 게 아니라, MetaMask가 변경 시 이벤트로 알려준다.

## STEP 8 — 프론트엔드: API 호출과 화면 렌더링

모든 API 호출은 동일 패턴: `로딩 → fetch('/api/...') → res.json() → DOM 업데이트 / catch → 에러 표시`.

```js
// 가격 조회 — 코인별 USD/KRW, 24h 변동률, 시가총액 표시 + 플래시 효과
async function fetchPrices() {
  var data = await (await fetch('/api/prices')).json();
  coins.forEach(function (coin) { /* data[coin.id]로 테이블 행 생성 */ });
}
// setInterval은 STEP 9 DOMContentLoaded에서 등록

// 블록 정보 — timestamp는 Unix초 → ×1000으로 JS Date 변환
async function fetchBlockInfo() {
  var data = await (await fetch('/api/block-info')).json();
  document.getElementById('block-timestamp').textContent =
    new Date(data.timestamp * 1000).toLocaleString('ko-KR');
}

// TX 조회 — 해시로 상세 정보 표시
async function searchTransaction() {
  var hash = document.getElementById('tx-hash-input').value.trim();
  var data = await (await fetch('/api/tx?hash=' + encodeURIComponent(hash))).json();
}

// ETH 전송 (POST) — 주소+금액 → 서버 전송 → TX해시, Etherscan 링크 표시
async function sendETH() {
  var res = await fetch('/api/send-eth', {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ to: toAddress, amount }),
  });
}
```

> [!note] 정석
> `fetch()`는 브라우저 내장 HTTP 클라이언트. 같은 서버 `/api/*` 호출이므로 CORS 문제 없음(Same-Origin). POST는 `method`, `headers`, `body` 명시 필수.

## 부록 — 핵심 개념 사전

### 이더리움 & 블록체인

| 용어 | 설명 | 비유 |
|------|------|------|
| **Wei / Gwei / ETH** | 최소 / 가스 단가 / 기본 단위. 1 ETH = 10^18 Wei | 전 / 원 |
| **Gas** | EVM 연산 비용. 수수료 = gasUsed × gasPrice | 자동차 연료 |
| **Nonce** | 계정 TX 순번, 리플레이 방지 | 거래 일련번호 |
| **Block** | TX 묶음. ~12초마다 생성 | 장부 한 페이지 |
| **Validator** | 32 ETH 예치 후 블록 생성 (PoS) | 공증인 |
| **Mempool** | 블록 미포함 TX 대기열 | 우체국 분류함 |
| **ECDSA** | 타원곡선 디지털 서명 (secp256k1) | 위조 불가 도장 |
| **Chain ID** | 네트워크 식별 (Mainnet=1, Sepolia=11155111) | 국가번호 |

### ethers.js

| 용어 | 설명 | 비유 |
|------|------|------|
| **Provider** | 블록체인 읽기 전용 연결 | 읽기 전용 전화선 |
| **Signer / Wallet** | 서명 객체 / 개인키 내장 Signer | 도장 달린 전화선 |
| **BrowserProvider** | MetaMask를 감싸는 Provider | MetaMask 통역사 |
| **formatEther / parseEther** | Wei↔ETH 변환 함수 | 단위 환산기 |

### 네트워크 & 프로토콜

| 용어 | 설명 |
|------|------|
| **JSON-RPC** | JSON으로 원격 함수 호출. 이더리움 노드 표준 통신 |
| **CORS** | 브라우저의 타 도메인 API 호출 제한 보안 정책 |
| **Infura** | 클라우드 이더리움 노드 서비스 (NaaS) |

| **EIP-1102** | DApp이 지갑 계정 접근 권한을 요청하는 메커니즘 |
