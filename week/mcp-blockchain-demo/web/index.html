MCP Client / Server Demo

날씨에서 코인까지, Tool 호출 흐름을 한 화면에서.

수업용 UI입니다. 버튼을 누르면 브라우저가 웹 백엔드에 요청하고, 백엔드는 MCP Client로 동작해 Weather/Coin MCP Server의 tool을 호출합니다. 교수님께는 “서버만 만든 게 아니라 client가 tool discovery와 call을 수행한다”는 점을 보여주면 됩니다.

실제 MCP stdio 연결 Open-Meteo API CoinGecko API 학생 기말 프로젝트 예시

작동 흐름

1
브라우저 UI도시/코인 입력 후 실행
2
Web Backend요청을 받아 MCP client 생성
3
MCP Servertool 목록 제공 및 tool 실행
4
External APIOpen-Meteo / CoinGecko 조회
5
Tool Result결과를 UI에 표시

Weather MCP

날씨 예제로 MCP 구조 설명
DEMO 
Weather MCP 결과가 여기에 표시됩니다.

예상 흐름:
MCP Client → get_current_weather(city) → Weather MCP Server → Open-Meteo API

Coin MCP

기말 프로젝트 방향 예시
PROJECT 
Coin MCP 결과가 여기에 표시됩니다.

예상 흐름:
MCP Client → get_coin_price(symbol) → Coin MCP Server → CoinGecko API
get_current_weather도시 현재 날씨
get_weather_forecast일별 예보
get_coin_price가격/시총/거래량
compare_coins여러 코인 비교
get_top_coins시총 상위 코인
교수님 설명 포인트: 이 화면은 단순 API 대시보드가 아니라 MCP client/server 구조를 UI로 감싼 것입니다. 브라우저가 직접 CoinGecko나 Open-Meteo를 호출하지 않고, 백엔드가 MCP client 역할을 하며 MCP server의 tool을 호출합니다. 학생들은 같은 구조에서 API와 tool 설계만 블록체인 도메인으로 확장하면 됩니다.
